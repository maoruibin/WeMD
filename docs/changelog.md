## 📋 版本规范

### 版本号规则

遵循 [语义化版本 2.0.0](https://semver.org/lang/zh-CN/)

```
MAJOR.MINOR.PATCH

MAJOR    # 不兼容的 API 变更
MINOR    # 向下兼容的新增功能
PATCH    # 向下兼容的 Bug 修复
```

### 发布流程

1. 从 `Unreleased` 移动已完成的变更到新版本
2. 更新发布日期
3. 添加详细变更内容
4. 创建 Git tag: `git tag v1.x.x`
5. 推送: `git push origin v1.x.x`
