# 版本发布清单

> 每次版本发布的检查清单

---

## ✅ 发布前检查

### 代码

- [ ] 所有 PR 已合并
- [ ] 版本号已更新 (package.json)
- [ ] CHANGELOG.md 已更新
- [ ] 构建测试通过
- [ ] 关键功能已测试

### 文档

- [ ] 版本文档已更新
- [ ] API 文档已更新
- [ ] 用户手册已更新

---

## 🚀 发布流程

### 1. 代码发布

| 平台 | 状态 | 备注 |
|------|------|------|
| Web 部署 | ⬜ | Vercel/自建服务器 |
| Chrome 插件 | ⬜ | 提交审核 |
| Android | ⬜ | Google Play |
| 桌面版 | ⬜ | GitHub Release |

### 2. 创建 Git Tag

```bash
git tag v1.x.x
git push origin v1.x.x
```

### 3. GitHub Release

- [ ] 创建 Release
- [ ] 填写 Release Notes
- [ ] 上传安装包
- [ ] 关闭相关 Issues

## 📊 发布后追踪

### 数据监控

发布后 24 小时关注：
- [ ] GitHub Stars 增长
- [ ] 下载/访问量
- [ ] 反馈评论
- [ ] Bug 报告

### 反馈处理

- [ ] 及时回复评论
- [ ] 记录用户反馈
- [ ] 评估 Bug 优先级
- [ ] 规划下一版本

## 🔄 回滚计划

如果出现严重问题：

1. 确认问题范围
2. 评估修复时间
3. 决定回滚或热修复
4. 通知用户
