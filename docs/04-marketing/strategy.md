# 运营策略

> WeMD 的运营方向和策略

---

## 🎯 运营目标

### 短期目标 (3个月)

- GitHub Stars 突破 500
- 日活用户 (DAU) 达到 100
- 完成首批 KOL 推广

### 中期目标 (6个月)

- GitHub Stars 突破 1000
- DAU 达到 500
- 公众号粉丝突破 1000

### 长期目标 (1年)

- 成为公众号创作者的必备工具
- DAU 达到 2000+
- 建立稳定的用户社群

---

## 📢 运营渠道

### 优先级 P0 (必做)

| 渠道 | 频率 | 内容 | 负责人 |
|------|------|------|--------|
| GitHub | 每次发布 | Release Notes | @me |
| 公众号 | 每周 1-2 篇 | 教程/更新 | @me |
| V2EX | 每次大版本 | 发布帖 | @me |

### 优先级 P1 (重要)

| 渠道 | 频率 | 内容 | 负责人 |
|------|------|------|--------|
| Twitter | 每周 2-3 条 | 动态/技巧 | @me |
| Product Hunt | 首次发布 | 发布页 | @me |
| 掘金/知乎 | 每月 1-2 篇 | 技术文章 | @me |

### 优先级 P2 (有时间做)

| 渠道 | 频率 | 内容 | 负责人 |
|------|------|------|--------|
| B站 | 每月 1 个 | 视频教程 | @me |
| Indie Hackers | 每月 1 篇 | 创业故事 | @me |
| 少数派 | 投稿 | 评测文章 | @me |

---

## 📝 内容策略

### 内容类型

| 类型 | 占比 | 主题 |
|------|------|------|
| 教程类 | 40% | 使用技巧、功能介绍 |
| 更新类 | 30% | 版本更新、新功能 |
| 故事类 | 20% | 开发故事、用户故事 |
| 观点类 | 10% | 行业观点 |

### 内容日历

```
周一：功能介绍/教程
周三：用户故事/案例
周五：开发日志/更新预告
```

---

## 🚀 发布策略

### 版本发布节奏

| 版本类型 | 频率 | 渠道覆盖 |
|----------|------|----------|
| Major | 年度 | 全渠道 |
| Minor | 季度 | 主要渠道 |
| Patch | 按需 | GitHub + 公众号 |

### 发布清单

每次发布前检查：
- [ ] 更新 changelog
- [ ] 准备截图/GIF
- [ ] 写发布文案
- [ ] 安排发布时间
