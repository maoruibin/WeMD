# WeMD 产品价值观

> 指导产品决策的核心原则

---

## 🧭 核心价值观

### 1. 本地优先

**用户的数据属于用户**

- 默认使用本地存储（IndexedDB、File System Access API）
- 不强制登录即可使用核心功能
- 云端同步作为可选项，而非必需

### 2. 简洁高效

**减少 friction，让创作流畅**

- Markdown 语法，不被格式干扰
- 快捷键支持，减少鼠标操作
- 一键复制，无缝对接微信

### 3. 所见即所得

**预览即实际效果**

- 98%+ 还原微信深色模式
- 支持微信特有的样式元素
- 实时预览，即时反馈

### 4. 尊重用户

**不绑架，不广告**

- 不收集用户写作内容
- 不在编辑器投放广告
- 开源透明，用户可控

## 🎯 产品原则

### 功能原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 少即是多 | 每个功能都要有明确价值 | 不做富文本模式 |
| 渐进增强 | 核心功能简单，高级功能可选 | 基础主题免费，高级主题付费 |
| 专注极致 | 在一个领域做到最好 | 专注微信排版，不做全能编辑器 |

### 设计原则

| 原则 | 说明 |
|------|------|
| 内容为王 | 界面简洁，突出内容 |
| 暗色友好 | 完美支持深色模式 |
| 响应迅速 | 快速加载，即时响应 |

### 技术原则

| 原则 | 说明 |
|------|------|
| 现代化 | 使用最新稳定技术 |
| 可维护 | 代码清晰，文档完善 |
| 可扩展 | 插件化设计 |

## 🤔 决策框架

当面临产品决策时，问自己：

1. **是否支持本地优先？**
   - 否 → 重新考虑
   - 是 → 继续

2. **是否让创作更简洁？**
   - 否 → 砍掉
   - 是 → 继续

3. **是否在微信场景下有价值？**
   - 否 → 不做
   - 是 → 评估优先级

4. **是否与现有功能冲突？**
   - 是 → 整合或取舍
   - 否 → 可以加入

## 📋 优先级判断

| 优先级 | 标准 | 示例 |
|--------|------|------|
| P0 | 核心体验，不做影响大 | 深色模式预览 |
| P1 | 重要功能，用户常提 | 图片上传 |
| P2 | 锦上添花，有时间再做 | 快捷键自定义 |
| P3 | 不确定价值，先观察 | 社交分享 |
