## 📋 支持流程

### 问题处理流程

```
1. 收集问题
   ↓
2. 分类诊断
   ├─ Bug → 记录到 Issues → 排期修复
   ├─ 建议 → 记录到 Backlog → 评估优先级
   └─ 疑问 → 直接解答 → 更新 FAQ
   ↓
3. 反馈用户
   ↓
4. 关闭/跟踪
```

### 优先级判断

| 优先级 | 定义 | 响应时间 |
|--------|------|----------|
| P0 | 崩溃、数据丢失 | 4h |
| P1 | 核心功能异常 | 24h |
| P2 | 次要功能问题 | 3天 |
| P3 | 体验优化 | 1周 |

## 📚 常见问题 (FAQ)

### 使用问题

**Q: 是否支持多端同步？**
A: 当前支持本地存储，云端同步规划中。

**Q: 支持导出哪些格式？**
A: 支持 HTML（复制到微信）、Markdown 原文。

**Q: 深色模式预览准确吗？**
A: 基于微信官方算法，98%+ 还原度。

### 技术问题

**Q: 数据存储在哪里？**
A: 默认使用浏览器 IndexedDB，完全本地存储。

**Q: 是否开源？**
A: 是的，完全开源，可自部署。

## 📝 反馈管理

### 反馈收集

- GitHub Issues

### 反馈处理

1. **确认**: 收到后 24h 内回复
2. **评估**: 3 天内给出处理方案
3. **排期**: 确认后加入版本计划
4. **通知**: 完成后告知用户
