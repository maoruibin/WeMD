## 🎯 背景与目标

### 用户痛点
- 用户无法在应用内直接查看当前版本号
- 无法快速访问项目官网、GitHub 仓库
- 项目品牌展示不足，缺乏存在感

### 目标用户
- 所有 WeMD 用户
- 想要了解项目信息的用户

### 目标
- 在侧边栏底部展示项目信息和版本号
- 提供快速访问官网、GitHub 的入口
- 增强品牌识别度

---

## 📖 功能描述

### 用户故事
**作为** WeMD 用户
**我想要** 在侧边栏看到项目信息和快捷链接
**以便于** 了解当前版本并快速访问相关资源

### 核心功能点
- [ ] 在文件侧边栏底部固定显示信息区域
- [ ] 显示当前版本号（如 v1.2.0）
- [ ] 显示项目 Slogan："Markdown 写作，一键到公众号"
- [ ] 显示快捷链接：官网、GitHub、关于文档
- [ ] 点击链接在新标签页打开

---

## 🎨 交互流程

### 正常流程
1. 用户打开应用，侧边栏底部显示信息区域
2. 用户点击"官网"链接，跳转到 https://weimd.gudong.site/
3. 用户点击"GitHub"链接，跳转到 https://github.com/tenngoxars/WeMD
4. 用户点击"关于"按钮，显示关于弹窗

### 交互说明
- 底部信息区域固定在侧边栏底部
- 文件列表滚动时，底部信息保持固定
- 链接在新标签页打开

### UI 草图

```
┌─────────────────────────────┐
│  WeMD 侧边栏                │

│ 📄 文章1.md                 │

│ ...                         │

├─────────────────────────────┤
```
